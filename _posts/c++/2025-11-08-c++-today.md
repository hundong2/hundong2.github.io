## 오늘의 C++ 최신 기술 트렌드: **Modern CMake를 활용한 빌드 시스템 구축 및 관리**

**1. 간단한 설명:**
Modern CMake는 CMake 3.0 이후의 버전을 사용하며, 기존의 CMake 사용 방식에서 탈피하여 보다 선언적이고 유지보수가 용이한 빌드 시스템을 구축하는 것을 목표로 합니다.  Modern CMake는 target 기반의 빌드 시스템을 구축하고,  `find_package`, `target_include_directories`, `target_link_libraries`, `target_compile_features`, `target_compile_definitions` 등의 명령어를 활용하여 헤더, 라이브러리, 컴파일 옵션 등을 모듈화하고 관리합니다. 이를 통해 프로젝트의 복잡성을 줄이고, 플랫폼 독립적인 빌드 시스템을 구축할 수 있습니다. Modern CMake는 또한 C++20 모듈을 지원하며, 더욱 강력한 빌드 도구로 발전하고 있습니다. Modern CMake를 사용하면 프로젝트 구조가 더 명확해지고, 빌드 스크립트가 더 간결해지며, 프로젝트의 의존성 관리가 더욱 쉬워집니다.

**3. 간단한 코드 예시 (C++):**

다음은 간단한 Modern CMake 예시입니다. 프로젝트 구조는 다음과 같습니다.

```
myproject/
├── CMakeLists.txt
├── include/
│   └── myproject/
│       └── mylib.hpp
└── src/
    ├── main.cpp
    └── mylib.cpp
```

`include/myproject/mylib.hpp`

```cpp
#ifndef MYLIB_HPP
#define MYLIB_HPP

int add(int a, int b);

#endif
```

`src/mylib.cpp`

```cpp
#include "myproject/mylib.hpp"

int add(int a, int b) {
    return a + b;
}
```

`src/main.cpp`

```cpp
#include <iostream>
#include "myproject/mylib.hpp"

int main() {
    std::cout << "1 + 2 = " << add(1, 2) << std::endl;
    return 0;
}
```

`CMakeLists.txt`

```cmake
cmake_minimum_required(VERSION 3.15) # 권장 버전 이상의 CMake 사용
project(MyProject VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17) # C++17 표준 사용
set(CMAKE_CXX_STANDARD_REQUIRED ON) # C++ 표준 준수 강제

# 헤더 파일 위치 설정
include_directories(${CMAKE_CURRENT_SOURCE_DIR}/include)

# 라이브러리 생성
add_library(mylib src/mylib.cpp)

# 실행 파일 생성
add_executable(myproject src/main.cpp)

# 라이브러리 링크
target_link_libraries(myproject mylib)

# 설치 규칙 추가 (선택 사항)
install(TARGETS myproject DESTINATION bin)
install(TARGETS mylib DESTINATION lib)
install(DIRECTORY include/ DESTINATION include)
```

**4. 코드 실행 결과 예시:**

1.  프로젝트 루트 디렉토리에서 빌드 디렉토리 생성: `mkdir build && cd build`
2.  CMake 실행: `cmake ..`
3.  빌드: `cmake --build .` 또는 `make` (Unix-like 시스템)
4.  실행 파일 실행: `./myproject` (build 디렉토리 내에서 실행)

출력:

```
1 + 2 = 3
```

Modern CMake는 프로젝트의 크기가 커질수록 그 장점이 더욱 부각됩니다. 이를 통해 빌드 시스템을 체계적으로 관리하고, 다양한 플랫폼에서 일관된 빌드 환경을 유지할 수 있습니다.
